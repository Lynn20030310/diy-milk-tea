# 微信小程序迁移计划

## 页面

- `pages/index`: DIY 主流程，包含茶底、奶基底、甜度、冰量、小料选择。
- `pages/result`: 配方卡片、价格、热量、保存和分享。
- `pages/saved`: 已保存配方列表。

## 数据模型

```js
{
  tea: "jasmine",
  milk: "fresh",
  sweetness: 1,
  ice: "less",
  toppings: ["pearl"]
}
```

## 第一阶段

- 使用本地静态数据驱动菜单。
- 用 `wx.setStorageSync` 保存配方。
- 用 Canvas 生成分享海报。

## 第二阶段

- 接入云开发数据库保存菜单和用户配方。
- 增加门店、订单、支付模拟。
- 将推荐配方改成后台可配置。

## 设计原则

- 首页直接进入 DIY，不做营销落地页。
- 底部固定价格摘要，减少用户反复滚动。
- 每次选择都要即时反馈价格、热量和视觉杯身变化。
